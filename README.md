## Course Objectives

1. Equip students with a comprehensive understanding of IoT technology, including its architecture, enabling technologies, communication models, and protocols, to effectively design and deploy IoT systems.

2. Enable students to address IoT security, privacy, and vulnerability issues, and apply practical IoT solutions using platforms like Raspberry Pi for real-world applications such as smart cities.

## Syllabus

### Module 1 — Introduction to IoT Technology (9 Hours) | CO1

- Definitions and Characteristics of IoT
- IoT Architectural View (3-layer, 5-layer, CISCO 7-level)
- Physical Design of IoT
- Logical Design of IoT — IoT Functional Blocks
- IoT Communication Models
- IoT Enabling Technologies
- IoT Levels & Deployment Templates

### Module 2 — IoT and M2M (9 Hours) | CO2

- M2M — Definition and Characteristics
- Difference between IoT and M2M
- SDN and NFV for IoT
- Smart Objects: The "Things" in IoT — Sensors, Actuators, Smart Objects
- Sensor Networks — Wireless Sensor Networks (WSNs)
- Communication Protocols for Wireless Sensor Networks
- Connecting Smart Objects — Communication Criteria

### Module 3 — Unified Data Standards & Protocols (9 Hours) | CO3

- IEEE 802.15.4 — Physical Layer, MAC Layer, Uses, Future (802.15.4e and 802.15.4g)
- Modbus
- ZigBee — Architecture
- LoRaWAN — Standardization, Physical Layer, MAC Layer, Topology
- LTE-M, NB-IoT
- Network Layer — Next Generation IP-based Protocols

### Module 4 — Raspberry Pi, Security & Smart City (9 Hours) | CO4, CO5

- Raspberry Pi Interfaces (Serial, SPI, I2C)
- Raspberry Pi Interfacing and Programming using Python (LED, Switch, Sensor, Serial, SPI, I2C devices)
- IoT Privacy, Security and Vulnerabilities — Security Requirements, Threat Analysis, Security Tomography, Layered Attacker Model, Identity Management, Access Control, Secure Message Communication
- Smart City IoT Architecture — Street Layer, City Layer, Data Center Layer, Services Layer

## Module 1 Study Notes

The **Module1_Complete_Study_Notes.docx** is a self-contained student reference document. Students do not need to refer to any other textbook — all content is drawn from the prescribed textbooks with inline citations.

**Coverage:**
- Topic 1: IoT Definitions & Characteristics `[TB3, TB2, REF2]`
- Topic 2: IoT Architectural View — 3-layer, 5-layer, CISCO 7-level, Oracle, oneM2M `[TB3, TB2]`
- Topic 3: Physical Design — IoT node components, MCU, sensors, actuators, platforms `[TB3]`
- Topic 4: Logical Design — 7 Functional Blocks `[TB1, TB2]`
- Topic 5: Communication Models — Request-Response, Pub-Sub, Push-Pull, Exclusive Pair `[TB1, REF2]`
- Topic 6: Enabling Technologies — WSN, Cloud, Big Data, RFID, IPv6/6LoWPAN, Embedded Systems `[TB3, REF2]`
- Topic 7: IoT Levels 1–6 & Deployment Templates `[TB1]`
- **Examination Questions** — 20 Part A (3 marks) + 8 Part B (9 marks) at the end
